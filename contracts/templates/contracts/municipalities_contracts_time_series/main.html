{% blocktrans %} Relevant context: {% endblocktrans %}

  • {% blocktrans %} The database only started to be populated from 2008, when European Union law came into effect. To some extent, this explains the sudden increase during 2008-2009. {% endblocktrans %}
  • {% blocktrans %} However, this does not explain the increase in 2009 to values to above the average value of dates posterior to 2009, i.e., why number of signed contracts decreased after end of 2009? {% endblocktrans %}
  • {% blocktrans %} There were two municipal elections; one in 11-10-2009, the other in 29-09-2013. In both cases, the number of contracts significantly increases before the election month. {% endblocktrans %}
  • {% trans "Expand technical details..." %}
  • {% blocktrans %} Because the database contains only a percentage of all contracts (not every contract is published), one important assumption is that this percentage does not depend on the signing date. (For experts:) the assumption is that the database constitute an unbiased sample of all contracts. {% endblocktrans %}
  • {% blocktrans %} We are not using the sum of values (â‚¬) of the contracts. The argument is that values are unbounded: one single contract can make a particular month to have a arbitrary large value. (For experts:) the reason is that the variance of the sum of values can be unbounded because the distribution of values is broad. {% endblocktrans %}
  • {% blocktrans %} On the other hand, the number of contracts is bounded. Moreover, the number of contracts reflects better the idea of "number of individual actions" the municipality does. {% endblocktrans %}
  • {% blocktrans %} There is a periodic peak of contracts in September. The beginning of school can be a justification for it. {% endblocktrans %}

{% blocktrans %} Overall, we argue that municipal elections have a strong influence on the number of contracts on months preceding the election. {% endblocktrans %} {% blocktrans %} In the particular case of 2009, this quantifies to an almost duplication of the number of contracts in relation to posterior dates. {% endblocktrans %}
